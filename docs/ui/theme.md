# UI 主题覆盖

本文件是项目内 UI 配色与视觉语义的事实来源，供 [static-ui skill](../../skills/static-ui/SKILL.md) 优先读取。
布局和 utility 写法约束见 [Utility 落地规则](utility-authoring.md)；本文件只描述当前实现已经落地的主题事实。

- 适用范围：`static/admin/`、`static/public/` 及共享静态样式。
- 当前不是“全站统一单一深色主题”。
- 当前主题基线是：
  - 后台：浅底玻璃面板 + 蓝绿品牌强调
  - 候选人端：深色答题壳层 + 蓝绿高亮与进度反馈

## 共享品牌事实

- 主品牌蓝：`#2563eb`
- 主品牌绿：`#16a34a` / `#22c55e`
- 品牌观感仍是“蓝色 + 绿色”
- 品牌色主要用于主操作、激活态、进度条、图标点缀和局部光晕，不用于整页大面积实底

## 后台主题

后台以 `static/assets/css/admin/theme.css` 和 `static/admin/index.html` 为准。

### 总体风格

- 明亮、通透、偏控制台感的浅色界面
- 页面底层是蓝绿冷色渐变，不是纯白平底
- 主要容器使用高透明白面板、细蓝色边框和较轻阴影

### 核心颜色

- 页面背景：`#f4f8ff` 到 `#eef5ff / #f5fbf7` 的浅色渐变
- 主文字：`#10233f`
- 面板背景：`rgba(255, 255, 255, 0.9)` 到 `rgba(255, 255, 255, 0.97)`
- 边框：`rgba(37, 99, 235, 0.16)`
- 主按钮 / 主导航激活：`bg-blue-600 text-white`
- 强调辅助色：`#16a34a`

### 组件基线

- 侧栏、头部、浮层都应维持浅色玻璃面板风格
- 非激活态导航优先使用白底 + 深色文字 + 轻蓝 hover
- 不要把后台整体改回深色卡片堆叠式控制台

## 候选人端主题

候选人端以 `static/public/index.html`、`static/public/views/*.html` 的模板 utility 为主，`static/assets/css/public/theme.css` 只提供辅助变量与基础渐变。

### 总体风格

- 深色答题壳层
- 蓝绿光晕背景
- 白字和浅灰辅助字
- 交互强调色集中在青绿进度、按钮和焦点态

### 核心颜色

- 页面主底：`bg-slate-950`
- 主文字：`text-white`
- 次文字：`text-slate-200 / text-slate-300 / text-slate-500`
- 卡片背景：`bg-white/5`、`bg-white/6`、`bg-slate-950/45`、`bg-slate-950/55`
- 卡片边框：`border-white/10`
- 主操作按钮：`bg-emerald-400 text-slate-950`
- 焦点 / 输入强调：`border-emerald-300/40`
- 进度与剩余时间：`teal / emerald`

### 说明

- 候选人端当前实际视觉是深色壳层优先，而不是复用后台的浅色玻璃面板
- 新增候选人端页面时，应优先延续现有深色答题体验，不要直接照搬后台浅色容器

## 状态色

- Danger：`rose`
- Warning：`amber`
- Info：`sky`

这些状态色继续保持语义独立，不和品牌蓝绿混用。

## 使用规则

- 新页面先判断属于后台还是候选人端，再继承对应壳层主题
- 品牌一致性来自蓝绿强调色，不来自“所有页面都必须同色底”
- 后台页面优先浅底白面板；候选人端优先深底高对比答题壳层
- 当 `skills/static-ui/SKILL.md` 的默认主题与当前实现冲突时，以本文件为准
