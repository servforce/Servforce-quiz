# Utility 落地规则

本文件定义项目内前端样式的落地约束，适用于 `static/` 下所有页面；未来如新增 `templates/`，同样遵守本规则。

## 核心原则

- 页面几何关系直接写在模板 utility 中，包括布局、断点、间距、宽高、对齐和显隐切换。
- 语义类只承载共享皮肤、共享机制和设计 token，例如统一表格滚动机制、导航外壳、状态变量。
- 同一个视觉结果只能有一个主导来源；不要让模板 utility 和 CSS override 同时控制同一层布局。

## 明确禁止

- 禁止选择器直接命中 Tailwind utility 名称，例如 `input.rounded-2xl`、`textarea.rounded-3xl`、`.px-4`。
- 禁止通过重复选择器提权压 utility，例如 `.foo.foo`、`.bar.bar.bar`。
- 禁止把页面级布局、间距、断点切换写进语义类媒体查询；页面骨架必须回到模板 utility。
- 禁止为了修单页问题，在共享样式文件里追加只服务于单页的补丁式 override。

## 允许的写法

- 允许定义语义类，但必须是可复用的共享能力，例如 `admin-data-table__scroller`、`admin-mobile-nav-shell`。
- 允许在共享类里定义视觉皮肤，例如边框、背景、阴影、blur、滚动行为、动画和 CSS 变量。
- 当某个页面需要特殊布局时，优先改模板里的 utility；只有抽象成多个页面复用的能力后，才提升为共享类。

## 评审检查项

- 是否存在 utility 选择器级重写。
- 是否存在 `.foo.foo` 这类提权写法。
- 页面断点布局是否能只通过模板 class 读懂。
- 删除某个页面专用 CSS 后，页面骨架是否仍然成立；若不能，说明布局没有回到模板。
