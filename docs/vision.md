# 项目愿景

## 项目定位

`md-quiz` 是一个基于 `Flask + PostgreSQL` 的在线笔试系统，面向招聘场景下的候选人邀约、身份验证、在线答题、自动判卷、简历处理和后台运营。

项目目标是把“试卷定义 -> 候选人分发 -> 验证入场 -> 在线答题 -> 自动判分 -> 结果归档 -> 后台回看”串成一条可持续运行、可维护、可观测的链路。

## 当前核心能力

- 支持管理员登录后台，管理试卷、候选人、分发记录、日志和系统状态。
- 支持使用 QML 风格 Markdown 上传试卷，并把题面、资源和公开邀请配置写入 PostgreSQL。
- 支持候选人专属邀约和公开邀约两种入口。
- 支持姓名 + 手机号校验，并可选接入阿里云短信验证码或本机号码认证。
- 支持在线答题、限时考试、最短交卷时长限制和自动收卷。
- 支持客观题自动判分，简答题通过 OpenAI-compatible Responses 接口调用 LLM 判分。
- 支持简历上传、文本提取、结构化解析和候选人画像补充。
- 支持后台 AI 辅助生成试卷，以及对生成结果做二次检查。
- 支持系统日志、每日指标、LLM/SMS 用量统计和告警状态查看。

## 当前范围边界

- 单组织后台，不做多租户、复杂权限模型和审计工作流。
- 默认优先服务招聘笔试场景，不扩展成通用 LMS 或考试 SaaS 平台。
- 异步能力以进程内后台线程为主，尚未拆成独立 worker / queue 系统。
- 数据权威源是 PostgreSQL；运行目录只保存上传文件和运行期产物，不承担业务真相源角色。
- LLM、短信和号码认证都是可选增强能力；未配置时系统仍应能完成基础考试流程。

## 当前不做的内容

- 多租户隔离、团队协作、组织级权限审批。
- 独立题库平台、复杂排考系统、课程化学习功能。
- 独立消息总线、分布式任务编排和多节点调度。
- 与招聘笔试无关的社交化、社区化或通用内容管理能力。
