## 4. 分阶段重构任务

建议顺序：

1. 收敛主入口与启动脚本
2. 收敛配置来源与配置说明
3. 抽离服务层，清理路由/脚本中的业务逻辑
4. 只有确认存在真实需求时，再拆异步执行面
5. 补齐关键测试与文档

每个阶段至少写清：

- 目标
- 具体改动
- 不改什么
- 验收方式

## 5. 验收标准

- 服务可按统一入口启动
- 主链路功能不回退
- 配置来源清晰且有说明
- 路由层、服务层、异步层职责可解释
- 文档与测试足以支撑后续维护
